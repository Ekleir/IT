Бывает __объект генератор__ и генератор коллекции.
Генератор - это подвид итератора. Генератор описывает протокол итерации.
Что бы создать генератор необходимо в функции вместо _return_ написать _yeld_
Создастся объект генератор, который будет выдавать значение по запросу, а не все сразу, как в _return_
![[Pasted image 20230705000108.png]]
Запрос значения происходит посредством передачи объекта генератора в функцию __next()__
Благодаря этой механике можно в функцию добавлять несколько __yeld__ и код будет выполняться до _yeld_ включительно, а при следующем вызове код дальше, если есть.

![[Pasted image 20230705001150.png]]

В генераторе нельзя вернуться к предыдущему значению, только если их отдельно сохранять.

Для генератора он сам для себя является итератором. Т.е. при вызове  метода __iter()__ на генераторе он вернёт сам генератор.
